O que a aplicação faz: 

Criar um cadastro de clientes que cada cliente tem vários contatos e mostrar esses contatos. 

Requisitos: 

Informações precisam ter relacionamento de um usuário para vários contatos. 
Entidade de Usuário
Entidade de contato
Precisa ter CRUD de usuário
Precisa ter CRUD de contato
Cada usuário precisa ter: Nome completo, email, telefone, e data do registro
Cada contato deve ter todos os campos do usuário

    Relatório em tela: 
        cada cliente tem um card com suas informações pré renderizadas
        ao clicar neste card abre uma nova rota com infos do cliente no início e outros cards com seus contatos.             

Regras de negócio: 

-Precisa ter autenticação
-Precisa ter testes
-Precisa de documentação de como iniciar, rodar e utilizar a aplicação. 
-Link do deploy do front. 

Tecnologias: 

Node com express para o back;
React no front;
Talvez alguma lib tipo tailwind;

Passos: 

Criar arquivo .env
Configurar a conexão com banco de dados
Ver se tem erros na conexão
Criar as models com os relacionamentos
Criar e rodar as migrations 
Criar os Controllers
Criar os services
Criar os Malwares

Criar o webApp 
Criar tela de cadastro 
Criar tela de login 
Criar componente de inicial do usuário
Criar componente de card de contatos
Criar componente modal para cadastrar contatos
